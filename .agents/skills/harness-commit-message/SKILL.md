# Harness Commit Message

Harness 일감 변경을 커밋할 때 아래 규칙을 적용한다.

## 제목 형식

커밋 제목을 정확히 다음 형식으로 작성한다.

```text
[일감id] 작업 내용 요약
```

- `일감id`에는 사용자 화면에 표시되는 8자리 일감 ID를 사용한다. 사용자가 전체 ID 사용을 명시한 경우에만 전체 ID를 사용한다.
- 작업 내용 요약은 실제로 완료하고 검증한 변경을 짧은 한국어 문구로 작성한다.
- 대괄호 앞에 타입이나 이모지를 붙이지 않는다.
- 제목 뒤에 상태, 에이전트 이름, 모델 이름을 덧붙이지 않는다.
- 여러 일감의 변경을 한 커밋에 섞지 않는다.

예시:

```text
[f7c682fd] 대시보드에 활성 모델 표시
```

## 커밋 절차

1. 현재 사용자 요청이나 일감 기록에서 일감 ID를 확인한다.
2. 스테이징할 차이를 검토하여 해당 일감의 변경만 포함한다.
3. 관련 검사를 실행하고 성공 여부를 확인한다.
4. 위 제목 형식으로 커밋한다.
5. 커밋 제목과 포함 파일을 다시 확인한다.
